# Task Archive (任务归档)

将 task.md 中已完成和非待执行的工作包/活动归档到独立文档，保持 task.md 精简。

## When to Use

- 用户说 "任务归档" / "归档"
- 用户说 "task archive" / "archive tasks"
- task.md 文件过长（单行过长 / 行数膨胀）需要清理时

---

## task.md 结构识别

task.md 节结构（按出现顺序，缺失的节跳过）：

- `# Task Overview` 标题 + `## 📊 快速概览`（含 **进度** 行 + **最近更新** 行 + 规划文档/预算）
- `## 📦 归档索引`（表格：日期 / 文件 / 摘要）
- `## 📝 最近活动`（表格：日期 / 活动描述，按时间倒序）
- `## 📋 待办工作包`（**历史结构，当前可能不存在**：WP 表格，含 ✅完成 / 🔄进行中 / 📋待执行 行）

> 结构演进：早期 task.md 含 `📋 待办工作包` 表格；当前版本改用 `📝 最近活动` 表 + `📊 快速概览` 进度行承载 WP 完成记录（单行进度串可极长）。本 skill 兼容两种结构。

---

## 核心流程

### Step 1: 分析 task.md

读取项目根 `task.md`，识别**可归档**与**必须保留**：

**可归档**：
- `📋 待办工作包` 表格中状态为 `✅ 完成` 的行（若该表存在）
- `📝 最近活动` 表格中**超过保留窗口**的旧活动行（默认保留最近 5 条，更旧的归档）
- `📊 快速概览` 的 **进度** 行中已完成 WP 的冗长详细描述（归档原貌，task.md 保留摘要）
- "历史工作包" / "实施阶段" / "依赖图" 等纯历史参考章节（若存在）

**必须保留**：
- `# Task Overview` 标题 + `📊 快速概览` 节（进度行可摘要化但节结构保留）
- `📦 归档索引` 表（只追加新行，不删旧行）
- `📝 最近活动` 表的最近 5 条（含表头/分隔行）
- `📋 待办工作包` 表中状态**不是** `✅ 完成` 的行（🔄进行中 / 📋待执行 **绝对不归档**）
- 综合发展规划文档引用链接

### Step 2: 生成归档文档

1. 当前日期 `YYYY-MM-DD`
2. 创建 `docs/archive/task-archive-YYYY-MM-DD.md`（目录不存在则先建）
3. 写入归档内容，保留原貌：

```markdown
# Task Archive — YYYY-MM-DD

> 自动归档生成，源文件：task.md
> 归档范围：<说明本次归档了哪些节/条目>

## 📊 快速概览 — 进度行（归档原貌）

<原进度行全文>

## 📝 最近活动 — 旧条目（归档原貌）

| 日期 | 活动描述 |
|------|----------|
<被归档的旧活动行，原样>

## 已归档工作包（若存在待办工作包表）

<按原格式保留 ✅ 完成 WP 详情>
```

### Step 3: 更新 task.md

1. `📊 快速概览` 进度行：若过长，替换为摘要（如 `N/N (100%) | 路线图完成 ✅（WP-XXX~YYY，详情见归档）`），详细原貌已进 Step 2 归档文件
2. `📝 最近活动` 表：删除超过保留窗口的旧活动行，保留最近 5 条（含表头/分隔行）
3. `📋 待办工作包` 表（若存在）：删除所有 `✅ 完成` 行；变空则保留表头并标注"无待办工作包"
4. 在 `📦 归档索引` 表的**首行数据位置**（最新日期在前）插入新行：
   `| YYYY-MM-DD | [task-archive-YYYY-MM-DD.md](docs/archive/task-archive-YYYY-MM-DD.md) | <X 个 WP / Y 条旧活动> 归档 |`

### Step 4: 验证

```bash
ls docs/archive/task-archive-YYYY-MM-DD.md   # 归档文件存在
grep "📦 归档索引" task.md                     # 归档索引节在
grep "📝 最近活动" task.md                     # 最近活动节在
# task.md 行数 / 最长行字符数 较归档前明显下降
```

> 若 task.md 含 `📋 待办工作包` 表，额外 `grep "📋 待办工作包" task.md`；当前结构无此表则跳过该检查（grep 返回非 0 不代表失败）。

---

## 注意事项

- **task.md 是 git tracked 工作包索引**（CLAUDE.md / `.gitignore` 明确：不得 `git rm` / 忽略）。归档是**移动详细内容**到 `docs/archive/`，task.md 保留精简骨架——绝非删除 task.md。
- **`docs/archive/` 必须进 git**：`.gitignore` 已含 `!docs/archive/` 例外（覆盖 `docs/*` 忽略规则），归档文件入 git 保留可追溯性。若该例外缺失，归档前先在 `.gitignore` 的 `docs/*` 段补 `!docs/archive/`。
- 归档累积：已有归档索引表只追加新行，不重建 / 不删旧行。
- 非完成态行（🔄进行中 / 📋待执行）绝对不归档。
- 单次归档聚焦一个日期；跨日历史分别归档。

## 输出报告格式

```markdown
✅ 归档完成

📦 归档文件: docs/archive/task-archive-YYYY-MM-DD.md
📊 归档条目: X 个已完成 WP + Y 条旧活动 + Z 个历史章节
📋 task.md 剩余: W 行（最长行 K 字符）

🛑 归档完成，等待您的下一步指示
```

## 职责

审查最终方案的整体质量，确认可执行性，给出最终评价。

## 任务清单

### Step 1: 文档质量审查

- [ ] 文档结构清晰，章节划分合理
- [ ] 语言准确，无歧义表述
- [ ] 数据引用有来源标注
- [ ] 图表/矩阵完整

### Step 2: 可执行性审查

- [ ] 架构方案有具体的模块划分和接口定义（非仅描述方向）
- [ ] 安全模型有技术方案（非仅列出需求）
- [ ] 路线图有优先级排序和时间估算
- [ ] 风险缓解措施可操作

### Step 3: 综合评价

- [ ] 给出最终评价（通过/需修改）
- [ ] 列出优点和改进建议
- [ ] 确认文档可作为后续执行的基础

## 关键文件

### 输入（读取）
- `docs/design/harness-universal-platform-final-design.md` — 最终设计文档
- WP-111-3-verify 的验证结论

## 验收标准

- [ ] 文档质量达到可执行标准
- [ ] 无未解决的矛盾或遗漏
- [ ] 方案可独立阅读，无需查阅原始报告

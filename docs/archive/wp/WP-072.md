## 背景

WP-069-3 校验发现 max_batch_size 不支持通过 get_config 覆盖，与 WP-068 设计文档要求不符。当前硬编码为 5，用户无法通过配置文件自定义。

skill.md 中已有 get_config 的使用模式（~L422）：
```
concurrency_config = get_config("agent_dispatcher.concurrency")
```

## 目标

增加 get_config 读取路径，允许用户通过配置文件自定义批量大小，默认值保持 5。

## 实施计划

### Step 1: 修改 max_batch_size 初始化

**文件**: `plugins/core/skill-agent-dispatcher/skill.md`
**位置**: 批量大小控制参数段 (~L330)

将：
```
max_batch_size = max_batch_size if 'max_batch_size' in locals() else 5
```

改为：
```
max_batch_size = get_config("agent_dispatcher.max_batch_size", default=5)
```

**注意**: get_config 调用应与现有 concurrency 配置读取模式保持一致（~L422）。

### Step 2: 确保恢复逻辑兼容

Phase 0 恢复逻辑中已有 `max_batch_size = state.get("max_batch_size", max_batch_size)`，确保优先级正确：
1. 首次运行: get_config 读取配置（或使用默认 5）
2. 恢复运行: 从状态文件恢复（覆盖 get_config 值）

需确认恢复行在 get_config 行之后执行，保持现有顺序即可。

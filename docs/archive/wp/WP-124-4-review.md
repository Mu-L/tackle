## 职责

审查迁移方案的完整性和回滚安全性，确认用户升级体验顺畅。

## 任务清单

### Step 1: 迁移方案完整性审查

- [ ] 审查迁移命令覆盖所有 v0.1.x 到 v0.2.0 的配置变更
- [ ] 审查迁移指南文档的步骤是否完整、可操作
- [ ] 确认迁移前置条件检查清单完整
- [ ] 确认已知限制文档化

### Step 2: 回滚安全性审查

- [ ] 审查备份机制是否可靠（原始配置完整备份）
- [ ] 审查回滚步骤是否可逆（回滚后状态与迁移前一致）
- [ ] 确认回滚过程中无数据丢失风险
- [ ] 确认回滚操作有明确确认提示

### Step 3: 向后兼容性审查

- [ ] 确认 v0.1.x plugin.json 在 v0.2.0 下仍可正常工作
- [ ] 确认 v0.2.0 新增字段有合理默认值
- [ ] 确认不升级迁移时，现有项目不受影响

### Step 4: 综合评价

- [ ] 给出最终评价（通过/需修改）
- [ ] 列出发现的问题和改进建议
- [ ] 确认迁移方案满足父工作包目标

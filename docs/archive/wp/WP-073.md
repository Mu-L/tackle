## 背景

WP-069 校验发现 global_pause_flag 未持久化。当用户发出 pause 指令后如果发生上下文压缩，暂停状态会丢失，恢复后可能继续创建新 Teamee。

**触发条件**: pause 指令 + 上下文压缩同时发生
**影响**: 可能创建预期外的新 Teamee（风险低但用户已决定修复）

## 目标

将 global_pause_flag 加入持久化 Schema，确保上下文压缩恢复后暂停状态不丢失。

## 实施计划

### Step 1: 初始化 Schema 增加字段

**文件**: `plugins/core/skill-agent-dispatcher/skill.md`
**位置**: 初始状态写入段 (~L339)

在 initial_state 对象中增加：
```
"global_pause_flag": False,
```

### Step 2: Phase B.5 写回增加字段

**文件**: `plugins/core/skill-agent-dispatcher/skill.md`
**位置**: Phase B.5 (~L406)

在写回对象中增加：
```
"global_pause_flag": global_pause_flag,
```

### Step 3: Phase C.5 写回增加字段

**文件**: `plugins/core/skill-agent-dispatcher/skill.md`
**位置**: Phase C.5 (~L492)

在写回对象中增加：
```
"global_pause_flag": global_pause_flag,
```

### Step 4: Phase D.5 写回增加字段

**文件**: `plugins/core/skill-agent-dispatcher/skill.md`
**位置**: Phase D.5 (~L614)

在写回对象中增加：
```
"global_pause_flag": global_pause_flag,
```

### Step 5: Phase 0 恢复增加字段

**文件**: `plugins/core/skill-agent-dispatcher/skill.md`
**位置**: Phase 0 (~L354)，在 current_batch 恢复行之后

增加：
```
global_pause_flag = state.get("global_pause_flag", False)
```

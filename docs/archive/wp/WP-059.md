## 目标

消除 `skill-batch-task-creator` 中连续三位编号的隐式假设，确保批量创建跨越 WP-999 时不会产生位数不一致问题。

## 问题分析

### 来源

- WP-055 兼容性分析报告第 3.3 节（Medium 风险）
- WP-056 验证报告（基本准确，行 289 描述可更精确）

### 问题点

1. **行 308 附近**: "获取起始 WP 编号" 步骤未说明编号位数无限制
2. **行 434 附近**: 批量文档路径 `docs/wp/WP-XXX.md` 未明确无位数限制
3. **行 471-503**: 批量报告模板区域暗示三位编号
4. **行 602**: 示例 `WP-040.md, WP-041.md` 暗示三位连续编号

### 风险

批量创建时若起始编号为 WP-995，会生成 WP-995 到 WP-1004 的混合位数编号。AI 可能因三位对齐习惯产生不一致的编号格式。

## 实施计划

### Step 1: 在 "获取起始 WP 编号" 步骤添加注释（行 308 附近）

在 `"获取起始 WP 编号" [shape=box]` 后添加说明：

```
   > **注意**: 编号无固定位数限制。批量创建时允许三位和四位混合
   > （如 WP-998, WP-999, WP-1000, WP-1001），AI 应使用实际编号，不做位数对齐。
```

### Step 2: 在批量文档路径说明处添加注释（行 434 附近）

在 `6. **写入工作包文档** - 批量创建 docs/wp/WP-XXX.md` 处添加注释：

```
   > `WP-XXX` 中的编号为实际数字，无固定位数。
```

### Step 3: 更新示例包含跨 999 场景（行 602 附近）

将示例 `AI: [写入 WP-040.md, WP-041.md]` 修改为：

```
AI: [写入 WP-040.md, WP-041.md]
  （如跨 999 边界则为 WP-998.md, WP-999.md, WP-1000.md）
```

### Step 4: 验证修改

读取修改后的文件，确认所有添加的注释正确且不影响原有内容。

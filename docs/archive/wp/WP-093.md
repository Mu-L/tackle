## 目标

创建 `types/` 目录，为核心模块提供 TypeScript 类型声明。约 67 公共 API 需声明，JSDoc 94.6% 覆盖率辅助推断。

## 实施计划

### Step 1: 创建类型目录

新建 `types/` 目录，按模块组织：
- `types/plugin-interface.d.ts`
- `types/event-bus.d.ts`
- `types/state-store.d.ts`
- `types/config-manager.d.ts`
- `types/plugin-loader.d.ts`
- `types/logger.d.ts`
- `types/harness-build.d.ts`
- `types/manifest-resolver.d.ts`
- `types/validator-pipeline.d.ts`
- `types/hook-dispatcher.d.ts`
- `types/index.d.ts` — 统一导出

### Step 2: 编写类型声明

- 从 JSDoc 注释推断类型
- 定义接口、类型别名、枚举
- 确保与实际代码一致

### Step 3: 配置 package.json

- 增加 `types` 字段指向 `types/index.d.ts`

## 关键文件

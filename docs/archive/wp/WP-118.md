### 问题分析

当前 E2E 测试（L4 层级）完全缺失（WP-110-2 维度评估）。无 CLI 子进程级端到端测试，init → build → validate 工作流无覆盖。这意味着：

1. CLI 入口点（`bin/tackle.js`）的集成正确性无法验证
2. 子命令间的数据流（init 生成 settings.json → build 读取 registry → validate 检查输出）无覆盖
3. 错误路径（无效 registry、缺失 plugin.json）无测试
4. global/local 安装模式差异无覆盖

## 目标

建立 CLI 子进程级端到端测试套件，覆盖 `init → build → validate` 标准工作流：

1. **E2E 测试框架** — 基于 `node:test` + `child_process.execSync` + `os.tmpdir()`
2. **核心工作流覆盖** — init → build → validate 完整流程
3. **错误路径覆盖** — 无效 registry、缺失 plugin.json 等异常场景
4. **CI 集成** — E2E 测试在 CI 中运行

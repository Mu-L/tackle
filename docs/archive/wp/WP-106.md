**D5 详细修正**:
- 原建议: conventional-changelog (更主流)
- 修正为: 手写简易版 (约 80 行 Node.js)
- 理由: 项目零外部依赖是重要设计约束，conventional-changelog 引入间接依赖。手写版从 git log 提取 conventional commit 并按 Keep a Changelog 格式输出。

- 命令文件导出函数接收 context 对象 (含 packageRoot, targetRoot, flags 等)

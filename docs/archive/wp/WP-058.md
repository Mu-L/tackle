## 目标

消除 `skill-task-creator` 中大量三位编号占位符的隐式暗示，确保 AI 在生成工作包时不会固定位数假设。

## 问题分析

### 来源

- WP-055 兼容性分析报告第 3.2 节（Medium 风险）
- WP-056 验证报告（确认行号和内容准确）

### 问题点

1. **行 344**: "获取当前工作包编号" 步骤未说明编号无位数限制
2. **行 388 附近**: 子工作包命名 `WP-XXX-1-impl` 未明确 XXX 为实际编号
3. **行 468-474**: 拆分报告模板中大量 `WP-XXX` 三位占位符
4. **行 607-707**: 父/子工作包文档模板中约 100 行三位占位符

### 风险

AI 从模板推断编号格式时可能形成 "三位数" 隐式假设，在编号超过 WP-999 时生成错误的文档编号。

## 实施计划

### Step 1: 在 "获取工作包编号" 步骤添加注释（行 344 附近）

在 `2. **获取当前工作包编号** - 读取 task.md 找到最大 WP 编号` 后追加说明：

```
   > **注意**: `WP-XXX` 中的 `XXX` 为数字编号占位符，无固定位数限制。
   > 编号范围涵盖 WP-1 到 WP-9999+，AI 应使用 task.md 中实际的最大编号 +1。
```

### Step 2: 在子工作包命名说明处添加注释（行 388 附近）

在 `WP-XXX-1-impl` 格式说明处添加注释：

```
   > **注意**: `XXX` 为实际数字编号（如 WP-56, WP-999, WP-1000），非固定位数。
```

### Step 3: 在父工作包文档模板添加占位符说明（行 607 附近）

在 `# WP-XXX: [任务名称]` 模板标题前添加一行注释：

```markdown
<!-- XXX = 数字编号占位符（如 WP-56, WP-1000），非固定位数 -->
```

### Step 4: 在拆分报告模板添加说明（行 460 附近）

在拆分报告模板的 `WP-XXX` 占位符旁添加类似注释。

### Step 5: 验证修改

读取修改后的文件，确认所有添加的注释正确且不影响原有内容。

## 背景

WP-069 综合校验报告发现 R1/R2 两个高风险问题：

- **R1**: Phase B.5/C.5/D.5 三处状态写回中 `current_batch` 硬编码为 `[]`，上下文压缩恢复后批次进度丢失
- **R2**: R1 触发后分批逻辑每轮重新执行，`pending_batches` 可能被重复追加

**触发条件**: 任务数 > max_batch_size 且上下文压缩发生在 Phase C 创建 Teamee 之后

## 目标

将 3 处硬编码的 `"current_batch": []` 改为使用实际变量值 `"current_batch": current_batch`，并在 Phase 0 恢复逻辑中补充 current_batch 字段恢复。

## 职责

审查 API 分类的合理性，确认 public / internal / experimental 的划分符合实际使用场景和演进计划。

## 任务清单

- [ ] 审查 public API 分类是否合理（是否真有外部消费者）
- [ ] 审查 experimental API 分类是否合理（是否确实可能变更）
- [ ] 审查 internal API 分类是否合理（是否仅内部使用）
- [ ] 检查分类标准是否一致（无同类 API 分到不同分类）
- [ ] 评估 API 稳定性承诺是否可持续（public 契约是否可维护）
- [ ] 审查 api-reference.md 文档质量

## 验收标准

- [ ] API 分类合理，符合实际使用场景
- [ ] 分类标准一致，无矛盾
- [ ] public API 契约可持续维护
- [ ] api-reference.md 文档质量良好

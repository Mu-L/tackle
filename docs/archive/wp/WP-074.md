## 背景

WP-069-3 校验发现 `unblocked_candidates` 未显式排序，依赖 `TaskList()` 返回顺序。不同运行轮次中 TaskList 返回顺序可能不一致，导致分批结果不稳定。

当前代码（~L427）：
```
unblocked_candidates = [t for t in tasks if t.status == "pending" and t.owner == "" and is_unblocked(t) and t.id not in teamee_map]
```

## 目标

对 unblocked_candidates 按 task ID 排序，确保相同任务集始终按固定顺序分批。

## 实施计划

### Step 1: 增加 sorted 排序

**文件**: `plugins/core/skill-agent-dispatcher/skill.md`
**位置**: 批量大小控制段 (~L427)

将：
```
unblocked_candidates = [t for t in tasks if t.status == "pending" and t.owner == "" and is_unblocked(t) and t.id not in teamee_map]
```

改为：
```
unblocked_candidates = sorted([t for t in tasks if t.status == "pending" and t.owner == "" and is_unblocked(t) and t.id not in teamee_map], key=lambda t: t.id)
```

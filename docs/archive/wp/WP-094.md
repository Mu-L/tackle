## 目标

基于 conventional commits 自动生成 Changelog。决策 D5 已确认：手写简易版 (约 80 行 Node.js)，因为项目零外部依赖是重要设计约束。

## 实施计划

### Step 1: 编写 changelog 生成脚本

新建 `scripts/changelog.js` (~80 行)：
- 解析 `git log` 输出
- 按 conventional commit type 分组 (feat/fix/docs/style/refactor/test/chore)
- 生成 Markdown 格式
- 保持与现有 CHANGELOG.md 格式一致

### Step 2: 集成到发布流程

- 可在 `publish.yml` 的 tag push 触发时自动运行
- 也可手动运行 `node scripts/changelog.js`

## 关键文件

- `scripts/changelog.js` — 新建
- `CHANGELOG.md` — 更新

## 验收标准

## 背景

WP-069-2 校验发现 skill.md 中 json_loads() 调用（~L292, ~L339）缺少 try-catch 容错。状态文件损坏时会导致解析异常，agent 停止运行。

**影响**: 状态文件由代码自身通过 json_dumps 写入，损坏概率极低。异常后 agent 停止而非产生错误行为（fail-safe）。但仍建议增加容错，fallback 到默认初始状态继续运行。

## 目标

在 Phase 0 恢复逻辑中为 json_loads 调用增加 try-catch 容错处理，异常时 fallback 到默认初始状态。

## 实施计划

### Step 1: 修改 Phase 0 恢复逻辑

**文件**: `plugins/core/skill-agent-dispatcher/skill.md`
**位置**: Phase 0 (~L354)

将：
```
state = json_loads(Read(file_path=state_file))
```

改为包裹 try-catch：
```
try:
    state = json_loads(Read(file_path=state_file))
except Exception as e:
    print(f"⚠️ 状态文件解析失败，使用默认初始状态: {e}")
    state = {}
```

并在后续的 `state.get(...)` 调用中确保 fallback 默认值正确（已有 `get` 的默认值机制，无需额外修改）。

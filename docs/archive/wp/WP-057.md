## 目标

消除 `skill-split-work-package` 中唯一显式的三位数字硬约束，确保 AI 在编号超过 WP-999 时不会拒绝生成四位编号。

## 问题分析

### 来源

- WP-055 兼容性分析报告（风险总览表 High 级别）
- WP-056 验证报告（确认行号和内容准确）

### 问题点

1. **行 352**: 显式声明 `WP-XXX (三位数字)` — 全部代码中唯一的三位数字硬约束
2. **行 353**: 前导零示例 `WP-001 → WP-002 → WP-003` — 强化固定位数暗示
3. **行 354**: 子任务编号 `GAZE-001, SHOP-002` — 前导零延伸到子任务
4. **行 371**: 模板 `XXX-001` — 前导零模板
5. **行 385-390**: 验收/测试 `XXX-001-A1`, `XXX-001-T1` — 前导零模板

### 风险

当编号超过 WP-999 时，AI 可能：
- 拒绝使用四位编号
- 尝试使用前导零导致无法表示
- 生成子包文档时产生编号冲突

## 实施计划

### Step 1: 修改编号规则表（行 352-354）

将：
```markdown
| 工作包 ID | `WP-XXX` (三位数字) |
| 顺序递增 | WP-001 → WP-002 → WP-003 |
| 子任务 ID | `模块前缀-序号` (如 GAZE-001, SHOP-002) |
```

改为：
```markdown
| 工作包 ID | `WP-NNN` (数字编号，无位数限制) |
| 顺序递增 | WP-1 → WP-2 → WP-3 |
| 子任务 ID | `模块前缀-序号` (如 GAZE-1, SHOP-2) |
```

### Step 2: 修改工作包模板中的前导零（行 361, 366, 371, 385-390）

将模板中所有 `XXX-001`、`XXX-001-A1`、`XXX-001-T1` 等前导零编号改为无前导零格式：
- `XXX-001` → `XXX-1`
- `XXX-001-A1` → `XXX-1-A1`
- `XXX-001-T1` → `XXX-1-T1`

### Step 3: 验证修改

读取修改后的文件，确认：
1. 不再包含 "三位数字" 字样
2. 不再包含前导零编号示例
3. 修改不影响其他内容

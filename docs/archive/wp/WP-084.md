## 目标

为全部 15 个 Skill 插件编写结构性验证测试，确保插件格式合规，建立插件质量基线。

## 实施计划

### Step 1: 创建测试文件

新建 `test/runtime/test-skill-structure.js`，使用 `node:test`：

```javascript
// 对每个 Skill 插件验证:
// 1. plugin.json 存在且包含必填字段 (name, version, type, description)
// 2. skill.md 存在且非空
// 3. skill.md 包含关键 Section (When to Use, Flow Diagram, Execution Steps)
// 4. plugin.json 的 type 字段为 "skill"
```

### Step 2: 批量验证

- 读取 `plugins/plugin-registry.json` 获取所有 type=skill 的插件
- 遍历每个插件目录执行结构验证
- 汇总结果输出

### Step 3: 确保未来新增 Skill 自动纳入

- 测试从 registry 动态读取插件列表
- 新增 Skill 无需修改测试即可自动验证

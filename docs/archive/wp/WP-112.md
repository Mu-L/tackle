### 问题分析

当前 Tackle Harness 的安全成熟度仅 1/5（WP-109/WP-110 共识）。`plugin-loader.js:462` 的 `require(indexJsPath)` 无沙箱隔离，外部插件可执行任意代码。v0.2.0 的目标是先实现基本的安全意识机制——"安装时确认 + 构建时警告"——不实现沙箱，让用户在知情前提下做出决策。

完整的安全模型（Worker Threads 沙箱 + 运行时权限校验）计划在 v0.3.0 实现（~240min），超出 v0.2.0 预算。

## 目标

在外部插件首次被加载时，让用户意识到安全风险并做出知情决策。实现"安装时确认 + 构建时警告"：

1. **tackle install 用户确认提示** — 当外部插件声明了 capabilities（非空）时，要求用户确认安装
2. **tackle build 外部插件来源警告** — 构建时对外部插件输出 sourceType 和 capabilities 警告
3. **plugin.json capabilities schema 字段** — 在 plugin-validator.js 中验证 capabilities 字段结构

## 任务清单

### Step 1: tackle install 用户确认提示 (~15min)

在 `commands/install.js` 中增加确认逻辑：

- [ ] 实现 `confirmInstall(pluginName, registryEntry)` 异步函数
- [ ] 读取 plugin.json 的 capabilities 字段
- [ ] capabilities 为空时自动通过（仅使用基础服务）
- [ ] capabilities 非空时显示每个 capability 及其风险等级
- [ ] 交互模式下使用 readline 等待用户输入 `[y/N]`
- [ ] 非交互模式下检查 `TACKLE_ASSUME_YES` 环境变量
- [ ] 用户拒绝时中止安装流程

### Step 2: tackle build 外部插件来源警告 (~10min)

在 `harness-build.js._buildPlugin()` 中增加来源警告：

- [ ] 检测 `entry.sourceType === 'npm' || entry.sourceType === 'local'`
- [ ] 输出外部插件来源警告日志
- [ ] 如果有 capabilities 声明，列出声明的 capabilities
- [ ] 如果无 capabilities 声明，提示"limited to basic services"
- [ ] core 插件（sourceType=core）不触发任何安全提示

### Step 3: plugin.json capabilities schema 字段验证 (~5min)

在 `plugin-validator.js` 中增加 capabilities 字段的结构验证：

- [ ] 实现 `validateCapabilities(capabilities)` 函数
- [ ] 定义已知 capability 列表: `['filesystem', 'network', 'child_process', 'env']`
- [ ] 对未知 capability 名称输出 warning（不阻断构建）
- [ ] 集成到现有 `validatePlugin()` 流程中
